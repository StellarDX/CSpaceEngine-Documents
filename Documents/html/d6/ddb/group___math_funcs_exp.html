指数函数，包含exp

以下内容为AI智能解析，仅供参考

exp(x) 实现了双精度浮点数（float64）的指数函数，即计算 \( e^x \)。最初记载于 GLibC 中，采用范围缩减（range reduction）和多项式逼近（polynomial approximation）技术，结合预计算表（lookup table）确保高效率和精度。以下是核心原理的解析：

1. 特殊情况处理

  • 极小值处理 ：当 \( x \) 极小（接近 0）时，直接返回 \( 1 + x \)（避免下溢）。
  • 无穷大处理 ：
  • \( x = +\infty \)：返回 \( +\infty \)。
  • \( x = -\infty \)：返回 \( 0 \)。
  • NaN 处理 ：若 \( x \) 是 NaN，返回 \( x \)（遵循 IEEE 754）。

2. 范围缩减（Range Reduction）

核心思想是将 \( e^x \) 分解为：

\[ e^x = 2^k \cdot e^r \]

其中 \( k \) 是整数， \( r \) 是余数（满足 \( |r| \leq \frac{\ln 2}{2N} \)， \( N = 128 \)）。

步骤：

  1. 计算k ：

    \[ k = \text{round}\left( x \cdot \frac{N}{\ln 2} \right) \]

    使用浮点技巧 z = x * InvLn2N + Shift（Shift = 0x1.8p52）后舍入，再减去 Shift 得到整数 \( k \)（kd）和索引 ki。
  2. 计算余数r ：

    \[ r = x - k \cdot \left( \frac{\ln 2}{N} \right) \]

    为减少误差，使用高低双精度乘法：
    r = x + kd * NegLn2hiN + kd * NegLn2loN; // 高精度计算

3. 查表获取 2^(k/N)

  • 查表索引 ：idx = 2 * (ki % N)（N = 128）。
  • 表结构 ：__Exp64f_table 存储预计算的 \( 2^{j/N} \)（ \( j = 0 \dots 127 \))，每项含两个 64 位整数：
  • table[idx] ：尾数部分（tail）。
  • table[idx + 1] ：指数基值。
  • 合成 scale ：
    sbits = table[idx + 1] + (ki << (52 - 7)); // 合并指数
    scale = __Float64::FromBytes(sbits); // 转为浮点数
    此时 scale 近似 \( 2^{k/N} \)。

4. 计算 e^r 的多项式逼近

因 \( r \) 很小，用泰勒级数近似 \( e^r \)：

\[ e^r \approx 1 + r + \frac{r^2}{2!} + \frac{r^3}{3!} + \frac{r^4}{4!} + \frac{r^5}{5!} \]

代码使用秦九韶算法（Horner 法）高效计算：

tmp = tail + r * (C1 + r * (C2 + r * (C3 + r * (C4 + r * C5))));

其中 tail 是查表值（补偿精度），系数 C1~C5 为 \( \frac{1}{n!} \)。

5. 合并结果

最终结果为：

\[ e^x = \text{scale} \times (1 + \text{tmp}) \]

实际计算避免额外舍入：

result = scale + scale * tmp;

6. 边界处理（特殊路径）

当 |x| 极大时：

  • k > 0（可能上溢） ：
    sbits -= 1009ull << 52; // 调整指数
    result = 0x1p1009 * (scale + scale * tmp);
  • k < 0（可能下溢） ：
    sbits += 1022ull << 52; // 调整指数
    y = scale + scale * tmp;
    // 精细处理下溢舍入
    result = 0x1p-1022 * y;

复数指数函数 expc

对复数 \( z = a + bi \)，利用欧拉公式：

\[ e^z = e^a \cdot (\cos b + i \sin b) \]

代码实现：

complex64 expc(complex64 z)
{
return exp(z.real()) * (cos(b) + 1i * sin(b)); // b = z.imag()
}

总结

  • 精度 ：通过范围缩减将问题限制在 \([- \frac{\ln 2}{256}, \frac{\ln 2}{256}]\)，结合 5 阶多项式逼近，确保高精度。
  • 性能 ：预计算表和 Horner 法减少计算量。
  • 健壮性 ：特殊值处理完备，避免上溢/下溢。

此实现是数值计算的经典实践，平衡了效率与精度。
